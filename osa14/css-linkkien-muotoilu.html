Linkin pseudoluokat

Linkillä on todellisuudessa neljä eri toimintavaihetta tai tilaa. CSS:llä linkin jokaisen vaiheen ulkoasu voidaan määrittää halutun kaltaiseksi. Tästä johtuen linkkien toimintaa ei voida määrittää pelkkää a-selektoria käyttämällä.

Linkkien ulkoasu määritellään kirjoittamalla CSS-tiedostoon jokaiselle vaiheelle omat määritykset. Eri vaiheiden erottaminen tapahtuu niin sanotuilla pseudoluokilla. Selektori ja siihen liitetty pseudoluokka kertovat, mitä vaihetta sääntö kuvaa. Selektori ja pseudoluokka erotetaan toisistaan kaksoispisteellä.

A-tagin neljä pseudoluokkaa ovat:

a:link - käymätön linkki
a:visited - käyty linkki
a:hover - kursori linkin päällä
a:active - aktivoitu linkki (hiiren nappi pohjassa)

Linkkien pseudoluokat pitää kirjoittaa CSS-tiedostoon aina yllä olevassa järjestyksessä. Muuten määritykset eivät toimi oikein. Eri selaimet reagoivat väärään järjestykseen eri tavoin. Vaikka haluaisit tehdä muutoksia vain yhteen vaiheeseen, tulee toimivuuden varmistamiseksi ehdottomasti määritellä aina kaikki linkin vaiheet.

Linkkien muotoilusta löytyy harjoituksia w3schools-sivuston CSS Links -sivuilta.

Suosituksia linkkien muotoiluun

Oletusmuodossaan käymättömät linkit ovat sinisiä ja alleviivattuja. Käydyt linkit taas ovat sinipunaisia ja alleviivattuja.

Tekstin seassa olevat linkit voi aivan hyvin antaa olla oletusmuodossaan. Jos muutat tekstin seassa olevien linkkien ulkoasua, määritä värit niin, että linkit varmasti erottuvat riittävän selvästi leipätekstistä ja taustasta. Eli reilu kontrasti linkin ja taustan välillä ja selvä väriero leipätekstin ja linkin välillä.

Navigointilinkeissä alleviivauksen poisto on yleinen käytäntö, ja se parantaa navigointijärjestelmän selkeyttä. Lisäksi navigointilinkeissä käydyn ja käymättömän linkin ulkoasu määritetään useimmiten samanlaisiksi, jotta navigointijärjestelmän ulkoasu pysyy yhtenäisenä sivustoa käytettäessä.

Esimerkkejä linkkien muotoilusta

Alla on kuvattu kaksi tyypillistä tapaa muotoilla leipätekstin seassa olevia linkkejä.

Esimerkki 1 - alleviivauksen poisto

Alleviivauksen poisto on yleinen keino parantaa linkkitekstien luettavuutta. Tässä esimerkissä alleviivauksen poisto tehdään link ja visited vaiheisiin. Hover ja active vaiheissa linkit näytetään alleviivattuina.

a:link {
text-decoration: none;}
a:visited {
text-decoration: none;}
a:hover {
text-decoration: underline;}
a:active {
text-decoration: underline;}

Esimerkki 2 - värin vaihto ja alleviivauksen poisto

Tässä esimerkissä käymättömien ja käytyjen linkkien väri on muutettu oletusväriä vaaleamman siniseksi. Hover- ja active-vaiheissa linkin väri muuttuu tummansiniseksi ja linkki muuttuu alleviivatuksi. Taustaväri on määritetty kaikissa vaiheissa läpinäkyväksi.

a:link {
text-decoration: underline;
color: blue; }
a:visited {
text-decoration: underline;
color: blue; }
a:hover {
text-decoration: none;
color: rgb(0, 132, 255); }
a:active {
text-decoration: none;
color: rgb(0, 132, 255); }
