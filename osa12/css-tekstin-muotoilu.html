Fontin valinta - font-family

Font-family-ominaisuus määrittää kirjasintyypin (fontin) jolla teksti näytetään. Yhdessä säännössä voidaan antaa useampia fontteja erottamalla ne toisistaan pilkulla. Selain näyttää tekstin aina ensimmäisen fontin mukaan, jonka se tunnistaa annetusta listasta. Jos selain ei tunnista yhtään annetuista fonteista, se käyttää jotakin muuta koneelta löytyvää annetun fonttiperheen jäsentä.

Fontti on syytä määritellä aina sekä halutuilla fonteilla sekä fonttiperheellä. Seuraava esimerkki on hyvin tyypillinen. Se määrittää kaiken p-elementtien sisään sijoitetun tekstin esitettäväksi ensisijaisesti Arial fontilla, toissijaisesti Helveticalla ja ellei näitä koneelta löydy, niin jollain muulla päätteettömällä eli sans-serif-fontilla.

p { font-family: Arial, Helvetica, sans-serif; }

Jos fontin nimessä on välilyönti, on fontin nimi kirjoitettava lainausmerkkien sisään:

p { font-family: "Times New Roman", Times, serif; }

Tekstin tyyli - font-style

Font-style-ominaisuus voi saada arvon normal, italic ja oblique. Normal näyttää tekstin ilman korostuksia. Italic näyttää tekstin kursivoituna. Myös oblique näyttää tekstin kursivoituna. Seuraava esimerkki kursivoi kaikki h2 tason otsikot:

h2 { font-style: italic; }

Fontin koko - font-size

Kirjasinkoko määritetään font-size-ominaisuuden avulla. Kirjasinkokoa voidaan määrittää usealla tavalla. Ylin esimerkki määrittää leipätekstin 16 pikselin korkuiseksi (yksikkönä px). Keskimmäinen esimerkki määrittää leipätekstin 1.1 kertaiseksi oletuskoosta (yksikkönä em). Alin esimerkki määrittää leipätekstin 90% kokoiseksi fontin oletuskoosta (yksikkönä prosentti).

p { font-size: 16px; }
p { font-size: 1.1em;}
p { font-size: 90%; }

Muistettavaa fonttien koosta ja luettavuudesta

Fonttityypit poikkeavat luettavuudeltaan toisistaan. Ruudulta luettaessa päätteettömät, eli sans-serif-fontit (esimerkiksi Verdana, Arial, Trebuchet MS), ovat yleensä helpompia lukea kuin päätteelliset, eli sans-fontit (esimerkiksi Georgia, Times New Roman).

Jos määrittelet fontin koon suhteellisesti (% tai em), muista huomioida määritysten yhteisvaikutus. Mitä esimerkiksi seuraava sääntö tekee jos käytät sivulla sisennettyjä listoja: p, li: {font-size: 0.8em; }

Fonttien oletuskoot poikkeavat toisistaan suuresti. Times New Roman on suhteellisen pientä. Arial on jonkin verran suurempaa kuin Times New Roman. Verdana taas on vielä Arialiakin suurempaa.

Rivin korkeus - line-height

Tekstin luettavuuden parantamiseksi käytetään usein maltillista rivivälin kasvattamista. Sopiva arvo löytyy todennäköisesti 120 - 150% välimaastosta. Seuraava esimerkki suurentaa tekstikappaleiden rivivälin 140% suuruiseksi oletuskokoon nähden:

p { line-height: 140%; }

Fontin paksuus - font-weight

Font-weight ominaisuus määrittää fontin lihavuuden. HTML-korostustageilla (strong, em) teksti voidaan ainoastaan lihavoida tai kursivoida. Font-weight ominaisuudella fonttia voidaan sen sijaan sekä paksuntaa että ohentaa. Yleisin käyttökohde on HTML-otsikkotagien tekemän automaattisen lihavoinnin poisto. Automaattilihavointi saadaan poistettua arvoilla lighter tai 100.

Fonteilla ei todellisuudessa ole yleensä kuin 2 tai korkeintaan 3 paksuutta. Sopiva paksuus löytyy vain kokeilemalla.

  • 100-900
  • bold lihavoi (700)
  • normal esittää alkuperäisenä (400)
  • lighter ohentaa

Tekstin tasaus - text-align

Text-align-ominaisuudella voidaan määrittää tekstikappaleen vaakasuuntainen tasaus. Oletuksena selaimet tasaavat tekstin vasemmalle. Jos tasausta ei haluta muuttaa, ei tasaustapaa tarvitse määrittää.

  • left tasaa vasempaan reunaan (oletus)
  • right tasaa oikeaan reunaan
  • center tasaa keskelle
  • justify tasaa molempiin reunoihin

Leipätekstin tasausta molempiin reunoihin (justify) tulee välttää, vaikka se tekeekin sivusta näennäisesti huolitellun oloisen. Koska webissä tekstiä ei tavuteta juuri koskaan, tasaus tapahtuu pelkästään sanavälejä muuttamalla. Kun sanavälit vaihtelevat, tekstin luettavuus heikkenee selvästi. Myös oikeaan laitaan ja keskelle tasaamista tulee välttää lähes aina.

Kapiteelit - text-transform

Text-transform -ominaisuudella voidaan muuttaa teksti näytettäväksi kapiteelikirjaimilla. Koska kapiteelit vaikeuttavat tekstin lukemista merkittävästi, ominaisuutta tarvitaan ja käytetään hyvin harvoin. Ominaisuudella on kolme mahdollista arvoa:

  • uppercase muuttaa kaiken tekstin kapiteeleiksi
  • lowercase muuttaa kaiken tekstin pieniksi kirjaimiksi
  • capitalize muuttaa sanojen alkukirjaimet kapiteeleiksi
