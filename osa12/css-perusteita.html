CSS - Cascading Style Sheets

Mitä CSS:llä tehdään?

Tyylisäännöstöillä (Cascading Style Sheets) määritellään html-dokumenttien esitysasua. Tyylisäännöstö sisältää erilaisia html-tageille (h1, ul, li, p, body, jne.) laadittuja määrityksiä. Määritykset koskevat tyypillisesti tekstin ulkoasua, sivustolla käytettäviä värejä ja elementtien asettelua sivulle.

Sivuston kaikki tyylimäärittelyt kirjoitetaan lähes aina erilliseen CSS-tiedostoon, jolloin koko sivuston ulkoasua voidaan hallita yhtä tiedostoa editoimalla. Tyylitiedosto tallennetaan tiedostonimi.css muotoon. Tyylimäärityksiä voidaan kuitenkin kirjoittaa myös suoraan html-tiedostoihin. Tämä on järkevää oikeastaan vain silloin, kun jollekin sivulle halutaan tehdä sivuston yleisestä tyylistä poikkeavia määrityksiä.

Tyylitiedoston linkittäminen HTML-dokumenttiin

Tyylitiedosto linkitetään HTML-dokumenttiin lisäämällä head-osioon (head-tagiparin väliin) link-tagi. Link-tagin href-attribuutin arvo ilmoittaa CSS-tiedoston sijainnin ja tiedostonimen, rel-attribuutti taas kertoo tiedoston suhteen html-dokumenttiin.

<link rel="stylesheet" href="tyylitiedosto.css">

Yhteen HTML-dokumenttiin voidaan tarvittaessa liittää useita tyylitiedostoja. Useiden tyylitiedostojen käyttö voi olla järkevää, jos määrittelyjä tulee hyvin paljon ja niitä halutaan ryhmitellä eri tiedostoihin. Useita tyylitiedostoja on mielekästä käyttää myös sellaisilla sivustoilla, joissa sivuston eri osiot ovat erilaisia ulkoasultaan.

Kuinka pääsen alkuun?

Tyylisäännöstöjä kirjoitettaessa on ehdottoman järkevää käyttää kunnollista CSS-tiedostojen kirjoittamiseen tarkoitettua ohjelmaa. Ne helpottavat kirjoittamista huomattavasti ja vähentävät virheellisten määritysten riskiä. Lisäksi ne sisältävät käteviä aputoimintoja kuten koodin automaattitäydennyksen ja värien valintatyökaluja.

Kun kirjoitat tyylisääntöjä, ajattele aina mitä tarkkaan ottaen olet tekemässä, ja pohdi mihin kaikkeen luomasi määritys vaikuttaa. CSS:n kirjoittaminen vaatii tietoista ajattelua ja tarkkuutta. Virheellisten tai keskenään ristiriitaisten määritysten selvittäminen voi olla hyvinkin työlästä.

Jokaisen määrityksen toimivuus kannattaa testata selaimessa heti sen kirjoittamisen jälkeen. Seuraava määritys kirjoitetaan vasta kun edellinen on todettu toimivaksi. Ja tietysti, CSS-tiedosto on aina tarkistettava CSS-validaattorilla. Nämä kaksi kysymystä kannattaa pitää koko ajan mielessä:

  1. Onko olemassa jokin yksinkertaisempi tai taloudellisempi tapaa tehdä tämä asia?
  2. Vaikuttaako tekemäni määritys vain siihen mihin haluan vai myös johonkin muuhun?

CSS-säännön syntaksi

Jokainen CSS-sääntö koostuu kolmesta osasta:

  • selektori määrittää mihin muotoilu kohdistuu (leipäteksti, otsikko, jne.)
  • ominaisuus määrittää mitä ominaisuutta muutetaan (väri, koko, jne.)
  • arvo määrittää millaiseksi ominaisuus muutetaan (sininen, 15 pikseliä, jne.)

CSS syntaksi: selektori { ominaisuus: arvo; }
Esimerkki: h1 { font-size: 120%; }

Muistilista syntaksista

  • ominaisuus-arvo parit, eli deklaraatiot, kirjoitetaan kaarisulkeiden sisään
  • kaarisulkeiden pikanäppäimet: Win: Ctrl + Alt + 7, Mac: Alt + Shift + 8
  • ominaisuuden jälkeen tulee aina kaksoispiste
  • jokainen ominaisuus-arvo pari päätetään puolipisteeseen
  • jos arvo koostuu useasta sanasta, kirjoitetaan arvo lainausmerkkien sisään, esim: "Times New Roman"
  • jos arvolla on mittayksikkö (%, px, em, jne.), se kirjoitetaan kiinni arvoon ilman välilyöntiä, esim: margin: 0px;
  • jos yksi sääntö halutaan kohdistaa useaan selektoriin, eri selektorit erotetaan toisistaan pilkulla: ul, ol { margin-top: 0px; }
  • jos annetaan useita selektoreja, eikä eroteta niitä pilkulla, kohdistuu määrittely jälkimmäisiin elementteihin, jotka ovat ensimmäisten elementtien sisällä
  • esimerkiksi tämä sääntö muuttaa pelkästään numeroidun listan (ol) rivit punaisiksi mutta ei muuta luetelmalistan (ul) rivien väriä: ul ol {color: red;}

Vinkkejä

  • yhdelle selektorille voidaan kirjoittaa useita ominaisuus-arvo -pareja (deklaraatioita)
  • luettavuuden helpottamiseksi jokainen ominaisuus-arvo -pari (deklaraatio) kannattaa kirjoittaa omalle rivilleen
  • CSS-editorien aputoiminnot helpottavat opiskelua ja nopeuttavat työtä
  • /* Näin kirjoitan CSS-tiedostoon kommentin */

Mihin CSS:llä voi vaikuttaa?

CSS:stä on tällä hetkellä olemassa neljä eri versiota. CSS1, CSS2, CSS3 ja CSS4. Versiot eroavat toisistaan ominaisuuksien määrän osalta. CSS1 on vanhin suppein ja CSS4 uusin ja monipuolisin. Kaikki nykyselaimet tukevat kattavasti CSS3 ominaisuuksia. Selainten CSS4 tuki ei sen sijaan vieläkään ole täydellinen.

Tässä muutamia yleisimpiä CSS2:een sisältyviä ominaisuuksia:

Asettelu

  • elementin leveys
  • elementin korkeus
  • elementin sijainti
  • marginaalit elementin ulkopuolella
  • täytteet elementin reunan ja sisällän välillä
  • elementin reunus (väri, paksuus, tyyli)

Tekstin ominaisuudet

  • fontti
  • fontin koko
  • korostukset (kursivointi, lihavointi, alleviivaus, yliviivaus)
  • tekstin riviväli
  • tekstin tasaus
  • kirjasinten väli

Värit

  • elementin väri ja taustaväri
  • sivun taustaväri
  • reunusten väri

Linkkien toiminta

  • käymättömän linkin ulkoasu
  • käydyn linkin ulkoasu
  • aktiivisena olevan linkin ulkoasu (kursori linkin päällä)
  • linkin alleviivaus / alleviivauksen poisto (eri vaiheista)
